What Is Bridge?

A non-technical overview for anyone — no crypto background required.

The one-sentence version: Bridge is a stablecoin infrastructure platform (acquired by Stripe for $1.1B) that lets any business move, store, convert, issue, and spend digital dollars through a single set of APIs — without needing to understand blockchains, manage wallets, or handle crypto compliance.

Think of it like a behind-the-scenes payments backbone. If Stripe made credit card payments invisible to developers, Bridge does the same thing for stablecoin payments. Your users never need to know they're using a blockchain.

Why Does This Exist?

Problem 1: Moving money globally is slow and expensive. International wire transfers take days and cost $25–50+ per transaction. Card payments charge 2–3% interchange. Correspondent banking adds multiple middlemen. Trillions of dollars move inefficiently through pipes built decades ago.

Problem 2: Stablecoins solve this, but are too hard to use. Stablecoins (USDC, USDT, etc.) already move $15+ trillion annually — settling in seconds for fractions of a penny. But integrating them requires blockchain expertise, wallet infrastructure, compliance tooling, and regulatory licensing that most businesses don't have.

Bridge connects these two problems into a solution. It abstracts away every piece of blockchain complexity and presents stablecoins as just another payment rail — one that's faster, cheaper, and globally accessible. Developers write a few lines of code. Bridge handles the blockchains, custody, compliance, and conversion behind the scenes.

What Can You Build With Bridge?

Send & receive payments: Move funds globally using fiat rails, stablecoins, or both. On-ramps (fiat → crypto), off-ramps (crypto → fiat), and crypto-to-crypto transfers — all through one API.

Virtual accounts: Instantly issue USD, EUR, and MXN deposit accounts for your users with local bank details (account/routing numbers, IBAN). Users fund via ACH, SEPA, or wire.

Custodial wallets: Create and manage digital asset wallets at scale. Bridge handles on-chain security, gas management, key custody, and all technical complexity.

Issue your own stablecoin: Launch a branded stablecoin in days via Open Issuance. Customize reserves (BlackRock, Fidelity, Superstate backing), earn rewards on reserves (3–4% from US Treasuries), and tap into shared liquidity across all Open Issuance coins.

Issue cards: Visa-powered virtual and physical cards that spend directly from stablecoin balances. Available in Africa, Latin America, the US, with EU and APAC coming 2026. No cross-border fees when users transact locally.

How Does a Typical Payment Work?

Example: USD → USDC on-ramp

1. Your user deposits USD via ACH to a Bridge-assigned virtual account → 2. Bridge receives the fiat and converts to USDC → 3. USDC is deposited to a Bridge-managed custodial wallet or sent to an external address → 4. Done. Your user sees a stablecoin balance. You see a webhook confirmation.

Or in reverse: user sends USDC → Bridge converts to EUR → Bridge sends EUR via SEPA to a European bank account. Cross-border payment settled in minutes instead of days.

API Design Principles

RESTful over HTTPS — all requests must use HTTPS; plain HTTP is rejected. Authentication via Api-Key header (HTTP Basic Auth, no username/password). Keys are shown only once at creation. Supports idempotency keys for safe retries. Paginated responses for list endpoints.

curl -X GET 'https://api.bridge.xyz/v0/customers' \ -H 'Content-Type: application/json' \ -H 'Api-Key: <your-api-key>'

Environments

Production: api.bridge.xyz — live money movement, real compliance checks.

Sandbox: Full test environment. API keys are scoped to individual developer accounts (not shared across org). Simulates KYC flows, transfers, and webhooks without moving real funds.

Webhook System

Bridge pushes real-time event notifications for: transfer state changes, customer KYC status updates, card transactions, wallet deposits, virtual account activity. Webhooks are configurable via API or dashboard. Essential for async flows like ACH settlement (which takes 1–3 business days).

Supported Currencies

Fiat: USD, EUR, MXN, BRL, ARS, GBP, and others via local rails.

Stablecoins: USDC, USDT, PYUSD, USDB (Bridge native), and any Open Issuance coin. Also supports BTC conversion.

Payment rails: ACH, Wire, SEPA, PIX, SPEI + on-chain transfers across multiple blockchains.

Transfers

One-time fiat or stablecoin transfers. Convert from source currency to destination currency (e.g. USDC → EUR, BTC → USD, USD → USDC). Supports on-ramps, off-ramps, and crypto-to-crypto flows. Transfers can originate from customer deposits, Bridge-managed wallets, or prefunded accounts.

Static Template Transfers

Reusable payment instructions for recurring transfers. A static template provides persistent deposit instructions. When funds arrive, Bridge auto-matches to the template and creates a new transfer instance. Ideal for recurring payroll or treasury operations.

Virtual Accounts

Fiat deposit addresses with real bank details (account/routing numbers for ACH, IBAN for SEPA). Each address is tied to a specific customer for reconciliation. Supported currencies: USD, EUR, MXN. Users can receive traditional bank transfers that Bridge automatically converts.

Prefunded Accounts

Preload funds to a Bridge-managed balance and use it for instant fiat off-ramps without waiting for deposit settlement. In practice, a prefunded account is a Bridge wallet configured for on_behalf_of off-ramps. Critical for payroll scenarios: one liquidity wallet dispersing ACH to many employees.

Liquidation Addresses

Link a blockchain address to a mapped fiat or crypto destination. When crypto arrives at this address, Bridge auto-converts and forwards to the final destination. Set-and-forget automation for recurring crypto-to-fiat conversion.

Fees & Precision

Bridge does not charge FX markups or cross-border fees on card transactions where the user transacts locally. Transfer fees are transparent and configurable. All monetary values use string-based precision to avoid floating-point rounding errors. Monthly invoices available in the dashboard.

USDB — Bridge's Native Stablecoin

USDB is Bridge's own USD-pegged stablecoin, designed for developers. Programmable, secure, with rewards built in. Businesses earn reserve yield for holding USDB balances. Used as default settlement coin across Stripe Stablecoin Financial Accounts (available in 101 countries).

Custom Stablecoins via Open Issuance

Launch a fully branded stablecoin in days, not months. Bridge handles: reserve management, security, GENIUS Act compliance, liquidity, and mint/burn infrastructure. Issuers customize: blockchain support, smart contract features, reserve allocation (cash vs. Treasuries ratio), and reward distribution.

Shared Liquidity Network

The key innovation: all Open Issuance stablecoins are interoperable 1:1 with each other. Phantom's CASH can swap instantly for USDH (Hyperliquid) or mUSD (MetaMask) — on-chain, permissionless, no intermediary. New coins launch into existing liquidity instead of starting from zero.

Revenue Model for Issuers

Reserve rewards: Reserves are invested in US Treasuries and cash equivalents earning 3–4%. Rewards flow to the issuer for originating the stablecoin. Interchange: If paired with Bridge Cards, every card swipe generates interchange revenue. No burn fees: Enterprises don't pay to redeem their own stablecoin.

Live Open Issuance Coins

CASH (Phantom) — 15M+ user wallet, consumer spending & DeFi. USDH (Native Markets / Hyperliquid). mUSD (MetaMask). Plus coins from Dakota, Slash, Lava, and Takenos. All interoperable within the network.

GENIUS Act Readiness

All Open Issuance coins are designed for compliance with the GENIUS Act (first US federal stablecoin law). Bridge handles: full reserve backing attestations, AML/KYC integration, transparent reserve reporting, and regulatory licensing. Issuers don't carry additional regulatory burden.

Security Model & Compliance Framework

Multi-layer security architecture spanning API access, custody, compliance, and infrastructure.

All requests over HTTPS (HTTP rejected). API key authentication via Api-Key header. Keys shown once at creation, never retrievable. Sandbox keys scoped per developer (not org-shared). Idempotency keys for safe retry of non-idempotent operations. Rate limiting and abuse detection.

Built-in customer onboarding with endorsement-based identity lifecycle. Customers must pass KYC (individuals) or KYB (businesses) before accessing services. Endorsements unlock specific capabilities: base (account creation), transfers (money movement), wallets (custody), cards (card issuing). Missing requirements surfaced via API (e.g., proof of address requests). KYC Links allow white-labeled onboarding flows.
4

Real-time sanctions screening against OFAC and international lists. Transaction monitoring for suspicious patterns. Travel Rule compliance (sender/receiver information exchange for qualifying transfers). Configurable compliance frameworks that adapt to new regulations without breaking user experience.

Visa network security standards. Virtual cards with instant provisioning. Push provisioning to Apple Pay and Google Pay. Programmable transaction controls and custom fees. Card programs leverage Bridge and Stripe's combined compliance, fraud detection, and bank/network relationships.

When crypto deposits require refund but no return policy is configured, Bridge marks them missing_return_policy. Configure a return policy via API or dashboard, and Bridge auto-initiates refund to sender's original address. Prevents funds from being stuck in limbo.

Every movement is recorded on-chain and mirrored in the orchestration layer. Automatic reconciliation between on-chain and off-chain ledgers. Timestamped history for all virtual accounts, static memos, transfers, and wallet activity. Receipt URLs generated per transfer for downstream auditing.

Custody of Funds — Who Holds What, When

How customer funds flow through Bridge's custody model, when Bridge is custodian vs. when funds leave Bridge's control, and the path toward qualified custodian status.

Key principle: Bridge operates a custodial model. When funds are held in Bridge-managed wallets or stablecoin financial accounts, Bridge (a Stripe company) acts as the custodian of your stablecoin balance. Fiat funds in transit are held by banking partners (Lead Bank). Stablecoin reserves are held by independent financial institutions (BlackRock, Fidelity, Superstate). Bridge is not currently a qualified custodian under SEC or banking definitions — but is actively pursuing that status via an OCC national trust bank charter.

Fund Lifecycle — Where Money Sits at Each Stage

Stage 1: Fiat Deposit (User → Bridge)

When a user deposits USD via ACH, wire, or SEPA into a Bridge virtual account, the fiat is received into Bridge Building Inc's bank accounts held at partner banks (Lead Bank and others). At this stage, fiat funds are held by the banking partner on behalf of Bridge.

Custody: Banking partner (Lead Bank) — FDIC-insured up to standard limits on the bank's balance sheet

Not yet on-chain. Not yet in user's wallet. Bridge is the money transmitter facilitating the transfer.

Stage 2: Conversion & Wallet Deposit (Bridge Processing)

Bridge converts fiat to stablecoins (USDC, USDB, or a custom Open Issuance coin) and deposits them into a Bridge-managed custodial wallet. Bridge directly states: "Bridge (a Stripe company) acts as the custodian of your stablecoin balance."

Custody: Bridge — holds stablecoin balances in custodial wallets using HSM/MPC key management. Private keys are managed by Bridge; users never touch them.

USDB specifics: Balances held as USDB are always backed 1:1 by cash and short-duration money market funds managed by BlackRock. The stablecoin sits in Bridge's custody; the reserves sit at BlackRock.

Stage 3: User Holds Balance (Steady State)

While funds sit in a Bridge custodial wallet or Stripe Stablecoin Financial Account, Bridge is the custodian. Users can view balances, send, receive, and spend (via cards), but they do not hold private keys.

Custody: Bridge — custodial control. Stripe's legal terms explicitly state: "Stripe does not convert, hold custody of, or transfer any Stablecoin as a component of the Stablecoin FA Services" — that function is performed by the "SFA Settlement Services Provider," which is Bridge Building Inc.

The legal distinction matters: Stripe facilitates the transfer of funds from your Stripe Account to Bridge Building Inc. Bridge Building Inc is the entity that actually custodies the stablecoins.

Stage 4: Withdrawal / Off-ramp (Bridge → User)

When a user withdraws to an external crypto wallet or off-ramps to fiat, funds leave Bridge's custody.

Crypto withdrawal: Stablecoins transfer to user's external wallet address. Custody transfers to the user (or their self-custodial wallet provider).

Fiat off-ramp: Bridge converts stablecoins → fiat and sends via ACH/Wire/SEPA to user's bank. Custody transfers to the receiving bank.

Once funds leave Bridge's system, Bridge has no further custody or control.

Is Bridge a Qualified Custodian?

Current Status: No — But Actively Pursuing Federal Charter

Today, Bridge Building Inc operates as a registered Money Services Business (NMLS #2450917) with state money transmitter licenses. This is not the same as a "qualified custodian" under SEC Investment Advisers Act rules (Rule 206(4)-2), which requires a bank, broker-dealer, or trust company charter.

In progress: On February 17, 2026, Bridge received conditional approval from the OCC (Office of the Comptroller of the Currency) to form Bridge National Trust Bank — a federally chartered national trust bank. If finalized, this charter would authorize Bridge to:

• Custody digital assets under direct federal supervision

• Issue stablecoins as a federally regulated institution

• Manage reserves with federal banking safeguards

• Operate orchestration and settlement services under OCC oversight

Important limitations: A national trust bank charter is not a full-service commercial bank charter. Bridge would not be able to: accept demand deposits, offer checking/savings accounts, make commercial loans, or access FDIC deposit insurance. It is limited to fiduciary functions: custody, issuance, reserve management, and settlement.

Timeline: The OCC has not announced when final approval will be granted. Bridge is the 6th crypto firm to receive conditional OCC trust charter approval, following Circle, Ripple, BitGo, Fidelity Digital Assets, and Paxos (all approved Dec 2025).

Privy Integration (Flexible Custody)

Since Stripe acquired Privy (June 2025), Bridge integrates with Privy's flexible custody framework. Developers can choose: Bridge as custodian, self-custody for users, or third-party custodian. The custody model can adapt over time per user, per jurisdiction.

This means a developer using Bridge + Privy could start users in Bridge custody and migrate advanced users to self-custody — without rebuilding the product.

Reserve Segregation

Stablecoin reserves are not held by Bridge. They are held by independent, regulated financial institutions (BlackRock, Fidelity, Superstate for Treasuries; Lead Bank for cash). This provides bankruptcy remoteness: if Bridge were to fail, reserves would still be accessible through the reserve partners.

USDB reserves specifically: 100% backed by cash and short-duration money market funds with BlackRock.

Key Risk: Custodial Gap

Until the OCC charter is finalized, Bridge operates as a money transmitter, not a bank or trust company. This means:

• No FDIC insurance on stablecoin balances

• Not a "qualified custodian" under SEC rules for investment advisers

• Institutional clients with fiduciary obligations may face compliance questions about using Bridge for custody

The OCC charter would close this gap if finalized.

No other stablecoin infrastructure provider has access to Stripe's merchant base — half the Fortune 100, millions of businesses, $1.4T in annual payment volume. Paxos doesn't offer orchestration. Brale doesn't offer cards. Bastion is still in private beta for several products. Coinbase comes closest but is exchange-first, not infrastructure-first.

Bridge's main competitive weakness vs. Bastion and Paxos has been regulatory depth. Bastion holds a NYDFS trust charter (existing qualified custodian). Paxos holds NYDFS + OCC + Singapore MAS. Bridge operated under state MTLs only — until the Feb 2026 OCC conditional approval. If finalized, the national trust bank charter would put Bridge on equal or superior regulatory footing.

Roles, Access Control & Responsibilities

How Bridge separates concerns between the platform, the developer (you), customers (end users), and reserve/compliance partners.

Key principle: Bridge operates a custodial, platform-managed model — not an on-chain smart-contract role system like decentralized protocols. Access control is enforced at the API layer with API keys, customer endorsements, and dashboard-managed admin permissions. The trust model is: developers trust Bridge (and Stripe) to custody funds, enforce compliance, and manage blockchain operations.

Platform Roles

1. Bridge / Stripe (Platform Operator)

The infrastructure provider. Operates the entire platform including blockchain nodes, custody systems, compliance engines, and fiat banking relationships.

• Manages: blockchain interactions, wallet key custody (HSM/MPC), gas fees, nonce management, transaction signing

• Enforces: KYC/KYB, AML screening, sanctions checks, Travel Rule, transaction monitoring

• Operates: reserve management for issued stablecoins (BlackRock, Fidelity, Superstate, Lead Bank)

• Provides: regulatory licensing (NMLS, state MTLs, EU registration), Visa card network relationships

• Does NOT: make business decisions for developers, control how developers price or distribute their products, share customer data across developer accounts

2. Developer (Your Organization)

You — the business building on Bridge. You control your integration, your customers, and your product experience.

• Controls: API keys (full account access), customer creation & onboarding, transfer initiation, wallet provisioning, card program configuration

• Configures: webhook endpoints, return policies, stablecoin parameters (if issuing), card fees & branding

• Manages: team admin permissions (via Account page in dashboard), sandbox vs. production environments

• Cannot: access other developers' data, bypass KYC requirements for customers, directly manage private keys or blockchain transactions, modify reserve allocations after issuance setup

Important: API keys grant full access to your account. They must be stored securely and never shared publicly. Sandbox keys are per-developer, not per-org.

3. Customer (End User)

The individuals or businesses your product serves. They interact with Bridge only through your application — Bridge is invisible to them.

• Has: a verified identity (KYC/KYB), one or more endorsements unlocking capabilities, optional wallets, optional cards

• Can: deposit/withdraw via assigned virtual accounts, hold stablecoin balances in custodial wallets, spend via issued cards, receive transfers

• Cannot: access the Bridge API directly, create their own API keys, manage other customers, bypass endorsement requirements

4. Stablecoin Issuer (Open Issuance User)

A developer who has launched their own stablecoin via Open Issuance. Has additional controls beyond standard developer access.

• Configures: blockchain support, smart contract features, reserve allocation (cash vs. Treasuries), reward distribution, open vs. closed loop

• Earns: reserve rewards (3–4% from Treasuries), interchange (if cards enabled), transaction fees (configurable)

• Controls: stablecoin branding, interoperability settings (opt-in to shared liquidity), distribution channels

• Cannot: directly access reserves (managed by BlackRock/Fidelity/Superstate/Lead Bank), modify other issuers' coins, bypass GENIUS Act compliance requirements

5. Reserve Partners (BlackRock, Fidelity, Superstate, Lead Bank)

External financial institutions that hold and manage the reserves backing issued stablecoins. Completely independent from Bridge's operational systems.

• Manages: investment of reserves in US Treasuries and cash equivalents, liquidity provisioning, attestation reporting

• Cannot: access Bridge platform systems, view customer data, initiate transfers, modify stablecoin parameters

Separation of concerns: Reserve management is fully segregated from platform operations. Even if Bridge's API layer were compromised, reserves are held at independent, regulated financial institutions.

How Roles Interact During a Transfer

Setup (one-time): Developer registers, creates API key, configures webhooks. Customer is created and passes KYC/KYB (endorsements approved). Virtual account or wallet provisioned.

During a transfer: Developer's app calls POST /v0/transfers with API key → Bridge validates customer endorsements → compliance engine screens transaction (AML, sanctions) → FX engine locks rate → source funds received (fiat rail or on-chain) → destination funds delivered → webhook fired to developer's app. The customer sees only what the developer's UI shows them. Bridge is invisible.

Admin Permission Management

Dashboard Team admin permissions are managed directly from the Account page in the Bridge Dashboard. No support ticket needed. Admins can invite team members, set permission levels, and manage access. Production API keys may be shared across the org; sandbox keys are per-developer.

Key Security Principles

Custodial trust model: Unlike decentralized protocols where smart contract roles enforce access on-chain, Bridge uses a traditional custodial model. You trust Bridge (backed by Stripe) to operate the infrastructure honestly. This is appropriate for regulated financial services where counterparty accountability matters.

Defense in depth: Multiple layers — API authentication, endorsement-based authorization, compliance screening, custody isolation, reserve segregation — ensure no single failure compromises the system.

Regulatory accountability: Bridge holds real licenses (NMLS, state MTLs, EU registration) with regulatory obligations. Stripe's public company-grade compliance standards apply. This is not pseudonymous DeFi — there is a legal entity accountable for operations.

Data isolation: Developer accounts are fully isolated. API keys, customer data, transfer records, and wallet balances are not shared across organizations. Sandbox and production environments are completely separate.
